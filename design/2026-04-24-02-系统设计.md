## 1. 设计哲学

### 1.1 核心理念
> **"把塔罗占卜的仪式感，用现代 Web 技术重新演绎。用户可以在三种不同的情感空间里，选择最贴合当下状态的那一种。"**

- **仪式感**：每次占卜是一段完整的沉浸式体验，而非简单的点击翻牌
- **可塑感**：通过主题切换，同一套塔罗逻辑呈现三种气质 —— 神秘 / 极简 / 自然
- **移动优先**：手机是主要场景，桌面是扩展体验
- **克制感**：动画丰富但不打扰信息阅读，解读部分排版优先

### 1.2 设计原则
1. **主题无关**：所有业务组件不硬编码颜色/字体/间距，全部读 design tokens
2. **沉默优先于喧嚣**：动画是烘托，不是主角
3. **每个动作都有反馈**：洗牌、切牌、选牌、翻牌，每一步都有视觉/音效回应
4. **信息分层**：关键词 → 核心含义 → 详细解读（层级展开）
5. **触控优先**：所有可点击区域 ≥ 44×44pt

## 2. 主题系统（Theme System）

### 2.1 三种主题总览

> 详细的视觉 Token、配色、字体、动效请见第 12 章「主题详细规范」。

| 主题 ID | 中文名 | 英文名 | 情感关键词 | 适用场景 |
|---------|--------|--------|----------|---------|
| `mystic` | 神秘暗黑 | Dark Mystic | 深邃 · 神秘 · 仪式 | 夜间深度占卜 / 仪式感拉满 |
| `minimal` | 现代极简 | Modern Minimalist | 清新 · 疗愈 · 专注 | 日常快问 / 工作间隙 |
| `nature` | 疗愈自然 | Healing Nature | 温暖 · 包容 · 安全 | 情绪低落 / 需要陪伴感 |

这些是**所有主题都遵守的通用规范**，主题只决定颜色/字体/纹理/动效强度，不改变这些基础骨架。

### 2.4 主题切换机制

#### 2.4.1 技术方案
```
┌───────────────────────────────────────────┐
│  <html data-theme="mystic | minimal | nature">
│    │
│    ▼
│  :root[data-theme="mystic"]  { --bg-base: #0A0B1E; ... }
│  :root[data-theme="minimal"] { --bg-base: #FAF8F5; ... }
│  :root[data-theme="nature"]  { --bg-base: #F5EEE3; ... }
└───────────────────────────────────────────┘
```

#### 2.4.2 切换流程
```
用户在设置页选择主题
   │
   ▼
Pinia settings.setTheme('minimal')
   │
   ▼
persist 到 localStorage
   │
   ▼
document.documentElement.dataset.theme = 'minimal'
   │
   ▼
CSS 变量变化 → 所有组件自动重绘
   │
   ▼
（可选）View Transitions API 平滑过渡
```

#### 2.4.3 防 FOUC（首屏闪烁）
在 `index.html` 的 `<head>` 里写内联脚本：
```html
<script>
  const saved = localStorage.getItem('lumina-theme') || 'mystic'
  document.documentElement.dataset.theme = saved
</script>
```

#### 2.4.4 主题预览
设置页每个主题卡片实时渲染一张示例卡牌 + 一个按钮 + 一行标题，用于直观预览。

### 2.5 组件主题化原则

**✅ 允许：**
- 组件内使用 CSS 变量（`var(--bg-surface)`）
- 组件接受 `variant` prop 但 variant 必须覆盖所有主题
- 动效参数通过 token 定义（如 `--motion-duration-base`）

**❌ 禁止：**
- 组件内硬编码颜色值（`background: #0A0B1E`）
- 组件内硬编码字体（`font-family: Cinzel`）
- 业务组件感知当前主题（`if (theme === 'mystic')`）—— 应通过 token 解耦

## 7. 核心模块设计

### 7.1 洗牌算法
使用 **Fisher-Yates 洗牌** + **加密级随机源**：

```ts
// composables/useShuffle.ts

export function useShuffle() {
  function shuffle<T>(arr: T[]): T[] {
    const copy = [...arr]
    for (let i = copy.length - 1; i > 0; i--) {
      const j = getRandomInt(i + 1)
      ;[copy[i], copy[j]] = [copy[j], copy[i]]
    }
    return copy
  }

  function getRandomInt(max: number): number {
    // 使用 Web Crypto API 保证高质量随机
    const buf = new Uint32Array(1)
    crypto.getRandomValues(buf)
    return buf[0] % max
  }

  function randomReverse(): boolean {
    return getRandomInt(2) === 0
  }

  return { shuffle, randomReverse }
}
```

```ts
export function flipCard(el: HTMLElement, onMidpoint: () => void) {
  const tl = gsap.timeline()
  tl.to(el, {
    rotateY: 90,
    duration: 0.3,
    ease: 'power2.in',
    onComplete: onMidpoint,  // 切换牌面

    duration: 0.5,
    yoyo: true,
    repeat: 1,

  return tl
```

```ts
import Dexie, { type Table } from 'dexie'
import type { DivinationRecord } from '@/types/history'

class LuminaDB extends Dexie {
  history!: Table<DivinationRecord, string>

  constructor() {
    super('lumina-tarot')
    this.version(1).stores({
      history: 'id, timestamp, spreadId',
    })
  }
}

export const db = new LuminaDB()

export function useHistoryDb() {
  async function save(record: DivinationRecord) {
    await db.history.add(record)
  }
  async function list(limit = 50) {
    return db.history.orderBy('timestamp').reverse().limit(limit).toArray()
  }
  async function remove(id: string) {
    await db.history.delete(id)
  }
  async function clear() {
    await db.history.clear()
  }
  return { save, list, remove, clear }
}
```

## 9. 动画设计规范

### 9.1 时长 token
```
instant:  80ms    极快，如点击反馈
fast:     200ms   按钮 hover、淡入
base:     400ms   卡片翻转单程、过渡
slow:     800ms   抛牌入位、页面切换
epic:     1600ms  洗牌序列、主视觉入场
```

### 9.2 缓动曲线
```
ease-out-quart        主入场（cubic-bezier(0.25, 1, 0.5, 1)）
ease-out-back         卡片落位（带一点过冲）
ease-in-out-expo      翻牌中段
linear                持续旋转 / 粒子
```

### 9.3 分层与叠加规则
- **Z 轴分层**：背景粒子 < UI 面板 < 卡牌 < 模态框 < toast
- **同层动画**：卡牌入场错开 80ms（stagger）
- **跨页过渡**：旧页淡出 300ms 重叠新页淡入 300ms（共 500ms）
