## 1. 项目概述

### 1.1 产品定位
Lumina Tarot 是一款"沉浸式神秘学占卜"Web App：

- **离线优先**：首屏加载后无需网络，所有数据、图片、音效全部本地化
- **视觉炸裂**：以"星空 / 银河 / 光粒子 / 3D 翻牌"为核心视觉语言
- **易用易懂**：新手友好的引导式占卜，内置 78 张韦特牌解读文本
- **私密安全**：历史占卜记录本地存储（IndexedDB），不上传任何数据

### 1.2 目标用户
- 塔罗爱好者（进行日常占卜、学习记牌）
- 神秘学内容创作者（截图素材、直播素材）
- 普通用户（解压、娱乐、自我觉察）

### 1.3 核心差异点
对比市面上常见的塔罗网站（多数是静态 HTML + 简单 JS）：

| 维度 | 传统塔罗站 | Lumina Tarot |
|------|-----------|--------------|
| 离线能力 | ❌ 必须联网 | ✅ PWA 离线可用 |
| 视觉层次 | 2D 平面图 | 3D 翻牌 + 粒子 + Shader |
| 交互动画 | 点击即翻 | 洗牌 / 切牌 / 抛牌 / 翻牌全流程 |
| 数据隐私 | 服务端存储 | 纯本地 IndexedDB |
| 可扩展性 | 固定页面 | 组件化，可扩展牌阵 / 主题 |

| 模块 | 功能 | 优先级 |

## 3. 技术可行性

### 3.1 核心能力清单

| 能力 | 浏览器原生支持 | 是否需要库 |
|------|---------------|-----------|
| 本地数据存储 | ✅ IndexedDB / localStorage | Dexie.js（轻量封装） |
| 离线运行 | ✅ Service Worker + Cache API | vite-plugin-pwa |
| 2D/3D 动画 | ✅ CSS Transform3d / WebGL | GSAP / Three.js |
| 粒子特效 | ✅ Canvas / WebGL | tsparticles |
| 手势交互 | ✅ Pointer Events | VueUse `useSwipe` |
| 音效 | ✅ Web Audio API | Howler.js |
| 随机数（洗牌） | ✅ Math.random / crypto | 无需 |

**结论：所有需求都能在现代浏览器纯前端实现，无任何后端依赖。**

### 3.2 数据规模评估（无硬上限，按体验优先）

- **牌面图片**：78 张 × 500KB（WebP 中品质） ≈ 40MB；高品质可达 80MB+
- **牌意文案**：78 张 × 正逆位 × 平均 200 字 ≈ 60KB JSON
- **音效**：洗牌音 + 翻牌音 + 背景音乐 × 3 主题 ≈ 10MB
- **主题资源**：3 套主题背景 / 纹样 / 装饰音 ≈ 5-15MB
- **策略**：**总量不做硬限制**。首屏仅加载当前主题必需资源，其他主题按需懒加载；通过 Service Worker precache 关键资源，通过 IndexedDB 持久化大型媒体（可选升级）

## 4. 技术栈选型（最终）

### 4.1 核心框架

| 层级 | 选型 | 理由 |
|------|------|------|
| 构建工具 | **Vite 5+** | HMR 快、原生 ESM、生态成熟 |
| 框架 | **Vue 3.4+** | Composition API + `<script setup>` 开发体验佳 |
| 语言 | **TypeScript 5+** | 类型安全，牌面数据强约束 |
| 路由 | **Vue Router 4** | 标准方案 |
| 状态管理 | **Pinia** | Vue 官方推荐，TS 友好 |
| 样式 | **Tailwind CSS 3.4+** | 原子化 CSS，spacing token 已统一（xs/sm/md/lg/xl/2xl） |
| UI 组件库 | **shadcn-vue** | source-copy 模式、HSL + CSS 变量主题，天然支持多主题切换 |

### 4.2 动画与视觉库（重点）

| 库 | 版本 | 用途 | 体积(gzip) | 选用理由 |
|----|------|------|-----------|---------|
| **GSAP** | 3.12+ | 主动画引擎：时间轴、翻牌、洗牌、页面过渡 | ~30KB | 业界动画标杆，时间轴语法强大，性能优秀 |
| **Three.js** | 0.160+ | 3D 卡牌翻转、星空银河场景 | ~150KB | 可选按需加载；提供 WebGL 级别视觉 |
| **tsparticles** | 3.x | 粒子背景（星空、光尘、光晕） | ~40KB | 配置化，免手写 Canvas |
| **@vueuse/motion** | 2.x | Vue 组件声明式微动画（按钮、卡片悬浮） | ~15KB | Vue 生态最佳微动画方案 |
| **lottie-web** | 5.x | 可选：复杂矢量动画（符文、占星符号） | ~60KB | 只在需要时懒加载 |
| **canvas-confetti** | 1.x | 翻牌高光、占卜完成的光爆效果 | ~7KB | 轻量即插即用 |

### 4.3 工具与数据

## 5. 离线方案

### 5.1 资源缓存策略（Workbox）

```
┌────────────────────────────────────────────────────┐
│ Request                                            │
├────────────────────────────────────────────────────┤
│ index.html          → NetworkFirst（版本更新优先） │
│ *.js / *.css        → StaleWhileRevalidate         │
│ /cards/*.webp       → CacheFirst（永久缓存）       │
│ /audio/*.mp3        → CacheFirst                   │
│ /data/tarot.json    → CacheFirst                   │
└────────────────────────────────────────────────────┘
```

### 5.2 首次加载策略

- **核心资源**（首屏 JS/CSS + 框架代码 + 78 张牌缩略图）→ precache
- **大图**（卡牌高清图）→ 按需加载 + 首次访问后预取空闲预热
- **音效**→ 用户交互触发时懒加载

### 5.3 数据持久化

| 数据类型 | 存储方案 |
|---------|---------|
| 占卜历史记录 | IndexedDB (Dexie) |
| 用户偏好（主题/语言/音效） | localStorage |
| 78 张牌静态数据 | 编译进 bundle（`/src/data/cards.ts`） |

## 6. 动画方案

### 6.1 核心动画场景

| 场景 | 技术方案 | 库 |
|------|---------|----|
| **星空背景** | WebGL 粒子，鼠标视差 | tsparticles |
| **洗牌动画** | 堆叠的卡片做随机打散 + 回合 | GSAP Timeline |
| **切牌动画** | 单指触控 / 拖拽切牌堆 | VueUse `useDraggable` + GSAP |
| **抛牌动画** | 卡牌从牌堆飞到指定牌阵位置 | GSAP + cubic-bezier |
| **翻牌动画** | CSS 3D Transform（rotateY 180°） | 原生 CSS + GSAP |
| **高级翻牌** | WebGL 布料或光影 shader | Three.js（可选） |
| **按钮微交互** | hover 光晕、press 回弹 | @vueuse/motion |
| **页面过渡** | 淡入 + 放大 + 星光溢散 | Vue `<Transition>` + GSAP |
| **解读展示** | 文字逐字浮现 + 金色下划线 | GSAP SplitText（简化版） |
| **占卜完成** | 金色粒子爆发 + 光环扩散 | canvas-confetti + GSAP |

### 6.2 性能保障

- **分层渲染**：静态层（UI）+ 动画层（Canvas/WebGL）分离
- **降级策略**：检测 `prefers-reduced-motion` 提供无动画模式
- **按需加载**：Three.js 仅在启用 3D 模式时动态 `import()`
- **GPU 加速**：核心动画使用 `transform` / `opacity`，避免 layout thrashing
- **帧率监控**：开发环境集成 `stats.js`

## 7. 资源方案

### 7.1 塔罗牌图片

塔罗牌的韦特牌版本（Rider-Waite）**原始版本已进入公共领域**（1909 年出版，版权早已过期），可合法免费使用。

**图片来源备选：**
1. Wikipedia Commons 的 Rider-Waite 全套高清扫描（公共领域）
2. 开源重绘项目：[Sergio-Ignacio-Lopez/Rider-Waite-Tarot](https://github.com/topics/tarot)
3. AI 重绘：使用 Midjourney / SD 生成原创风格（需商用许可）

**推荐方案：**
- MVP 阶段使用公共领域扫描版（合法 + 免费）
- 长期规划 AI 重绘统一风格（彰显品牌）

### 7.2 牌意文案

- 韦特牌的经典解读文本可从公共领域书籍整理（如 Waite 本人的《The Pictorial Key to the Tarot》）
- 以 JSON 结构存储：

## 8. 风险与挑战

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|-------|------|---------|
| 首次加载资源过大 | 中 | 中 | 分级加载 + 当前主题优先 + 其他主题懒加载 |
| 低端设备动画卡顿 | 中 | 中 | 降级策略 + `prefers-reduced-motion` + 主题内置"轻动效"开关 |
| iOS Safari PWA 限制 | 高 | 低 | 主功能不依赖 Push，Safari 限制可接受 |
| 牌意文案翻译质量 | 中 | 中 | 先用公共领域英文，再人工润色中文 |
| Three.js 首次加载大 | 中 | 低 | 动态 import + 可选 3D 模式 |
| 牌面图片版权 | 低 | 高 | 优先使用公共领域素材，明确来源 |
| 主题视觉一致性 | 中 | 中 | 统一 design tokens + 所有组件主题无关化 |
| 主题切换闪烁（FOUC） | 中 | 低 | 首屏注入当前主题 CSS 变量 + View Transitions API |
| 移动端真机适配差异 | 中 | 中 | iOS SE 375px / Android 360dp 最小测试基线 |
